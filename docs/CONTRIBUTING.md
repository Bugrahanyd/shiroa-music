# CONTRIBUTING to SHIROA

Thanks for contributing! Please follow these steps.

## Workflow
- Fork repository or branch from `develop`.
- Create branch: `feat/your-feature-name`.
- Make changes, keep commits atomic.
- Push branch and open a Pull Request to `develop`.
- PR should include: description, screenshots (if UI), test plan.

## Code standards
- TypeScript strict mode
- ESLint + Prettier formatting
- Unit tests for new logic, 80% coverage target for core modules

## Issue & PR template
- Use labels: `bug`, `enhancement`, `docs`, `help wanted`.
- PR reviewer: at least one approver required.

## Local setup
- See README for local dev environment setup steps (docker-compose).
